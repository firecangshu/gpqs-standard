# 贡献指南 / Contributing Guide

感谢你考虑为 GPQS 项目做出贡献！

Thank you for considering contributing to the GPQS project!

---

## 如何贡献 / How to Contribute

### 1. 报告 Bug / Report Bugs

如果你发现了 Bug，请创建一个 Issue，并包含以下信息：
- Bug 描述（中英双语）
- 复现步骤
- 预期行为 vs 实际行为
- 截图（如果有）

If you find a bug, please create an Issue with the following information:
- Bug description (bilingual: Chinese + English)
- Steps to reproduce
- Expected behavior vs actual behavior
- Screenshots (if any)

### 2. 提出新功能 / Propose New Features

如果你有新的想法，请创建一个 Issue，并包含以下信息：
- 功能描述（中英双语）
- 为什么需要这个功能？
- 如何实现？

If you have new ideas, please create an Issue with the following information:
- Feature description (bilingual: Chinese + English)
- Why is this feature needed?
- How to implement it?

### 3. 提交代码 / Submit Code

**步骤**：
1. Fork 这个仓库
2. 创建你的功能分支（`git checkout -b feature/AmazingFeature`）
3. 提交你的修改（`git commit -m 'Add some AmazingFeature'`）
4. 推送到分支（`git push origin feature/AmazingFeature`）
5. 创建一个 Pull Request

**Steps**:
1. Fork this repository
2. Create your feature branch (`git checkout -b feature/AmazingFeature`)
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`)
4. Push to the branch (`git push origin feature/AmazingFeature`)
5. Create a Pull Request

## 代码规范 / Code Standards

### 1. 语言 / Language

- **文档**：必须提供中英双语版本
- **代码注释**：推荐中英双语，至少英文
- **提交信息**：英文（符合 [Conventional Commits](https://www.conventionalcommits.org/) 规范）

**Documents**: Must provide bilingual versions (Chinese + English)
**Code comments**: Bilingual recommended, at least English
**Commit messages**: English (follow [Conventional Commits](https://www.conventionalcommits.org/) specification)

### 2. 格式 / Format

- **Markdown**：使用 UTF-8 编码
- **YAML**：使用 2 空格缩进
- **JSON**：使用 2 空格缩进
- **JavaScript**：使用 2 空格缩进

### 3. 测试 / Testing

- 所有新功能必须包含测试
- 所有测试必须通过

**All new features must include tests. All tests must pass.**

## 社区规范 / Community Standards

### 行为准则 / Code of Conduct

- 尊重所有贡献者
- 使用友好、包容的语言
- 接受建设性的批评

**Respect all contributors. Use friendly and inclusive language. Accept constructive criticism.**
